# Daily Post - 公众号文案生成器

根据项目最新动态，生成轻松随意风格的公众号文案（300-500字）。

## 执行流程

### Step 1: 收集素材
1. 读取当前项目的 CHANGELOG.md（取最新 1-3 条）
2. 读取 GitHub 最近的 commits（如果有 .git）
3. 识别今日/近期的主要更新

### Step 2: 生成文案
按以下模板结构输出，用口语化、轻松的语气：

```
📦 项目动态

[根据 CHANGELOG 自动生成，用轻松的语气改写]
- 具体做了什么，而不是技术术语
- 对用户有什么好处

---

💡 技术小记

[留空，提示用户填写]
> 今天踩了什么坑？学到了什么？随便写两句

---

📅 下期预告

[留空，提示用户填写]
> 接下来打算做什么？

---

💬 碎碎念

[留空，提示用户填写]
> 随便聊两句，增加人情味
```

### Step 3: 输出格式
直接输出可复制到公众号的纯文本，不要 markdown 代码块包裹。

## 风格指南

### DO
- 用「我」而不是「我们」
- 说人话，避免技术黑话
- 适当用 emoji 增加活泼感
- 把技术成果翻译成用户价值

### DON'T
- 不要用「优化」「重构」等程序员黑话
- 不要罗列 commit message
- 不要太正式，像发朋友圈一样

## 示例输出

```
📦 项目动态

这两天给 LovStudio 加了个新功能：翻译器现在会记住你用过的模型了，下次打开直接用，不用再选一遍。

另外发现了一个小 bug，支付宝支付暂时关了，等修好再开（反正也没人用...）

---

💡 技术小记

> [你的技术心得]

---

📅 下期预告

> [你的计划]

---

💬 碎碎念

> [随便聊两句]
```

## 注意事项
- 如果 CHANGELOG 没有新内容，提示用户「今天没有新的项目更新，可以写写技术心得」
- 生成的文案直接输出，方便用户复制
